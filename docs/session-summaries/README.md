# Session Summaries

Purpose: short, consistent handoff artifacts for cross-LLM continuity without duplicating docs.

## Rules
- One summary per session.
- Keep it brief; link to canonical docs instead of copying content.
- Use date-first filenames: `YYYY-MM-DD_agent_topic.md`.
- If a summary overlaps a sprint/status doc, link to it and avoid restating.

## Template
Use `SESSION_SUMMARY_TEMPLATE.md` in this folder.
